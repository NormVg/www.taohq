# Maya UI Source Map

This file is generated by `skills/maya-design/scripts/sync-source-references.mjs` from `src/runtime`.
Use it as the routing layer before opening exact component source.

## Source-First Workflow

1. Read `references/architecture.md` to choose where copied files should live in the target project.
2. Read `references/source/maya.css` when theme tokens, shadows, radii, or animations are needed.
3. Read only the exact component files needed from `references/source/components/`.
4. If a component imports a local composable or child component, copy that dependency too.
5. Keep the copied component source close to the original first, then adapt names, props, styling, or dependencies deliberately.

## Theme and Runtime Files

| File | Use when |
|---|---|
| `references/source/maya.css` | Project needs Maya tokens, theme variables, button/input/table surfaces, shadows, or motion variables. |
| `references/source/plugin.ts` | Project is a Nuxt module integration or needs to understand how the package registers components/styles. |
